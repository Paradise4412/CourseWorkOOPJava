# Приложение для учёта задержаний (отделение полиции)

Курсовая работа по дисциплине «Объектно-ориентированное программирование (Java)».

## Описание

Консольное приложение для ведения учёта задержанных лиц: регистрация, поиск, изменение и удаление записей, статистика по статусам. Данные хранятся в реляционной базе **H2** (файловая БД в папке `data/`).

## Технологии

- Java 17
- Maven
- JDBC
- H2 Database

## Структура проекта

```
src/main/java/ru/istu/police/
├── Main.java              — точка входа
├── model/                 — модель данных (DetentionRecord, DetentionStatus)
├── db/Database.java       — подключение к БД и создание таблиц
├── dao/DetentionDao.java  — операции с данными (CRUD)
└── ui/ConsoleApp.java     — консольное меню
```

## Назначение файлов и пакетов

Ниже — краткое описание, **за что отвечает каждый файл**, чтобы было проще защищать проект.

### Корень проекта

- `pom.xml`: конфигурация Maven — зависимости (H2), версия Java, сборка исполняемого JAR.
- `.gitignore`: исключает из git `target/`, `data/`, файлы IDE.
- `README.md`: описание проекта, запуск и структура.

### Исходный код (`src/main/java/ru/istu/police/`)

- `Main.java`: точка входа (`public static void main`). Запускает консольное приложение и корректно закрывает соединение с БД.

#### Пакет `model/`

- `DetentionRecord.java`: модель (сущность) одной записи о задержании. Содержит поля (ФИО, даты, причина, статус и т.д.) и `toString()` для удобного вывода в консоль.
- `DetentionStatus.java`: enum со статусами задержания (например, `DETAINED`, `RELEASED`, `TRANSFERRED`). Нужен для единообразного хранения/поиска статуса и для статистики.

#### Пакет `db/`

- `Database.java`: отвечает за подключение к H2 через JDBC и первичную инициализацию схемы (создание таблицы `detentions`, если её ещё нет). Также содержит метод `close()` для закрытия соединения.

#### Пакет `dao/`

- `DetentionDao.java`: слой доступа к данным (DAO). Содержит SQL-операции:
  - **Create**: добавление записи в таблицу
  - **Read**: получение всех записей, поиск по id/ФИО/статусу
  - **Update**: изменение записи
  - **Delete**: удаление записи
  - **Statistics**: подсчёт количества записей и количества по статусам

#### Пакет `ui/`

- `ConsoleApp.java`: консольный интерфейс (меню). Считывает ввод пользователя, валидирует данные, вызывает методы `DetentionDao` и печатает результат.

### Директории, которые появляются при работе

- `data/`: создаётся при первом запуске. Здесь хранится файл базы H2 (например, `data/detentions.mv.db`).
- `target/`: создаётся Maven при сборке. Содержит `.class` файлы и собранный `detention-registry-1.0.0.jar`.

## Запуск

Требуется установленные **JDK 17+** и **Maven**.

```bash
cd oopJava
mvn compile exec:java -Dexec.mainClass="ru.istu.police.Main"
```

Или сборка JAR и запуск:

```bash
mvn package
java -jar target/detention-registry-1.0.0.jar
```

## Поля записи о задержании

| Поле | Описание |
|------|----------|
| ФИО | Фамилия, имя, отчество задержанного |
| Дата рождения | Формат: `дд.мм.гггг` |
| Дата и время задержания | Формат: `дд.мм.гггг чч:мм` |
| Причина | Основание задержания |
| Статья УК | Необязательно |
| Сотрудник | ФИО дежурного сотрудника |
| Камера | Номер камеры (необязательно) |
| Статус | Задержан / Отпущен / Передан в СИЗО |
| Примечание | Дополнительная информация |

## Функции меню

1. Добавить запись о задержании  
2. Показать все записи  
3. Найти по ФИО  
4. Найти по статусу  
5. Просмотреть запись по номеру  
6. Изменить запись  
7. Удалить запись  
8. Статистика  
0. Выход  

## База данных

При первом запуске создаётся файл `data/detentions.mv.db`. Таблица `detentions` создаётся автоматически. Для просмотра БД можно использовать [H2 Console](https://www.h2database.com/html/download.html) с URL:

```
jdbc:h2:./data/detentions
```

Логин: `sa`, пароль: пустой.
